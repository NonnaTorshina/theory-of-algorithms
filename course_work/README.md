# Курсовая работа на тему "Задача коммивояжёра"

## Описание проекта

Этот проект предназначен для решения классической задачи коммивояжера
(TSP - Travelling Salesman Problem) с использованием муравьиного алгоритма 
(Ant Colony Optimization). 
Проект включает в себя desktop-приложение с графическим интерфейсом на PySide6 
и веб-приложение на Django для удаленного доступа, реализованные на Python с использованием современных фреймворков, технологий и архитектуры MVC (Model-View-Controller).

## Функциональные возможности
### GUI-приложение - PySide6
- интерактивное создание графа путём добавления точек на координатную плоскость
- визуализация вершин, рёбер и найденных маршрутов
- настройка параметров муравьиного алгоритма (количество муравьев, итераций, коффициентов)
- решение задачи коммивояжера с визуализацией оптимального маршрута
- сохранение и загрузка результатов расчетов
- просмотр истории выполненных расчетов

### Веб-приложение - Django
- удобный веб-интерфейс для решения задачи коммивояжера
- интерактивное поле для расстановки вершин графа
- настройка параметров алгоритма через веб-формы
- просмотр истории расчетов на отдельной странице
- валидация входных данных и обработка ошибок

### Возможности алгоритма (что происходит для решения задачи)
- реализация муравьиного алгоритма с настраиваемыми параметрами
- поддержка симметричной задачи коммивояжера
- автоматическое вычисление матрицы расстояний
- визуализация процесса работы алгоритма
- сохранение статистики и метрик производительности

## Технические особенности
- **Архитектура**: MVC(Model-View-Controller) с четким разделением ответственности
- **Графический интерфейс**: PySide6 (Qt для Python) с современным дизайном
- **Веб-фреймворк**: Django 4.2 с шаблонизацией и QRM
- **База данных**: SQLite с автоматическим созданием схемы
- **Алгоритм**: Муравьиный алгоритм(ACO) с настраиваемыми параметрами
- **Контейнеризация**: Docker и Docker Compose для кроссплатформенного развертывания
- **Тестирование**: pytest для комплексного модульного тестирования
- **Логирование**: встроенная система логирования работы алгоритма

## Требования
- Python 3.8
- PySide6
- Django 4.2
- SQLite3
- Docker & Docker Compose (для контейнеризации)
- pytest 

## Установка и запуск
### Локальная установка

1. Клонируйте репозиторий:
```bash
git clone <https://github.com/NonnaTorshina/theory-of-algorithms/tree/master/course_work>
cd course_work
```
2.Установите зависимости:
```bash
pip install -r requirements.txt
```
3. Запустите GUI-приложение:
```bash
python main.py
```
4. Запустите веб-приложение:
```bash
cd taskmanager
python manage.py runserver
```

### Запуск через Docker
```bash
docker-compose up --build
```
### Для запуска тестирования:
```bash
pytest tests/tests.py -v
```

## Использование
### Создание графа
1. В режиме добавления точек кликните в любом месте поля для создания вершин
2. Переключитесь в режим создания ребер для соединения вершин (необязательно, так как при решении задачи, пути между всеми городами выстроятся автоматически)

3. При создании ребра система запросит вес (расстояние между точками)

###Настройка алгоритма
1. Количество муравьев: Размер колонии (по умолчанию: 100)
2. Количество итераций: Число циклов алгоритма (по умолчанию: 20)
3. Коэффициент α: Влияние феромонов на выбор пути (по умолчанию: 1.5)
4. Коэффициент β: Влияние эвристической информации (по умолчанию: 1.2)
5. Коэффициент ρ: Скорость испарения феромона (по умолчанию: 0.6)
6. Интенсивность Q: Количество откладываемого феромона (по умолчанию: 10.0)

###Решение задачи
1. Убедитесь, что создано не менее 3 точек
2. Настройте параметры алгоритма при необходимости
3. Нажмите "Решить задачу коммивояжера"
4. Найденный оптимальный маршрут будет визуализирован цветной линией
